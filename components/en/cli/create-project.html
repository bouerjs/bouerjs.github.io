Creating a Project

The CLI allows you to generate two types of stating templates, blank and routing. Follow the instructions bellow to create a bouer project: bouer create new ˂project-name˃

Blank Project 


    bouer create new hello-world

By default Blank template is used in project generation, to choose another template we use the command -t

Using Existing Template 


    bouer create new hello-world -t routing

Using routing template it generate a Bouer App with routing already configured.

The command bouer newcommand has a number of options and you can explore them all by running: 


    bouer create -h
    # or
    bouer create -help
      

    Usage: bouer create [options] [command]

    Create a new Bouer.js project, component, or service

    Options:
      -h, --help                            display help for command

    Commands:
      new [options] ˂project-name˃          Create a new Bouer.js project
      component [options] ˂component-name˃  Creates a new component
      help [command]                        display help for command

Use the command -h to see what's the options each command provides

Install Dependecies

Bouer CLI already installs all the needed dependecies when the project is created, but if for some reason you need to re-install the dependecies it also allows you to do it. 


    bouer install
