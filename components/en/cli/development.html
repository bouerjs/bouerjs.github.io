Development

Bouer CLI uses Webpack to launch and build the projects, among the following presets @babel/preset-env to be able to process the files in your project.

HTML and Static Assets

The file index.html is starting point of the application, after the compilation, the main .js, containing all the logic of your application will be added automatically in this file.

All the assets that needs to be imported in the main file we just need to import, the base tag will resolve the path of the files according to the valie provided in the href attribute.

Working with Webpack

The CLI uses webpack to build the project, it combines all the .js|.ts files into one and keep the structure of other files, like .html .css or .scss, and the assets. The files are requested only when they need to be used (if prefetch == false, we can find it in Bouer Instance Properties - for global or Component Properties - for specific).

By default the CLI has it own webpack.config.js that is used to build and run the project, but it can be overwritten by creating a new webpack.config.js, in case of using both, by merging it, use the command --mix-config when starting your app. 


    bouer run --mix-config webpack.config.js

When using --mix-config it does not overwrite the existing defined properties, it just adds the containing fields. In case of array, like: plugins or rules the arrays will be concatenated.

Avoid repeating configuration, you may use bouer create config --preview to preview to cli configuration


    bouer create config --preview

If you want to create this configuration in you project, remove the --preview option. 


    bouer create config

Make sure to remove all the already defined config, in arrays, to avoid the same configuration to be added many times.
