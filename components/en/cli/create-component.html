Creating Components

The CLI allows you to create components into a bouer project, a component includes the following files extentions: [.html .ts .scss]. by using the parameter [-p, -path] it allows you to add a component into a specific path, otherwise the component will be created into the root folder src.

Create with no Path 


    bouer create component ˂component-name˃

Create with Path 


    bouer create component ˂component-name˃ -p components

The components generated by the CLI has exactly the same logic as it was explained in components and module.

Let's create About component, the structure will be: 


    bouer create component About -p components
      

    |_ 📁 app
      |_ 📁 components
      |  |_ 📁 about
      |     |_ 📄 about.html
      |     |_ 📜 about.ts
      |     |_ 📊 about.scss
          

    // about.ts
    import { Component } from "bouerjs";
    import html from "about.html"
    import style from "about.scss"

    export default class About extends Component {
      constructor(){
        super(html, [style]);
      }

      // Added after the component generation
      route = "/about";
    }

The files .html and .scss are basically empty that's why it will not be shown here.
