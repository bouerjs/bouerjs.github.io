Getting Started

The Bouer CLI is a command-line interface tool which allows you to scaffold, develop, deploy, and maintain Bouer applications directly from a command shell.

Bouer CLI is published on npm as bouer-cli package and includes a binary named bouerjs. Commands invoking bouer are using the Bouer CLI.

Prerequisites

  • Node.js - v^14.20.1 or newer
  • Text editor - One good option is Visual Studio Code
  • Terminal - Required for running Bouer CLI commands

Installing

To get started with Bouer CLI, you need to install it globally using npm. This allows you to access the CLI from anywhere on your system. 


    npm install -g bouer-cli

This will download and install Bouer CLI globally, making it available for use in your projects.

Verify the Installation

Once the installation is complete, you can check if everything is set up correctly by verifying the installed version. 


    bouer -v

    # or

    bouer --version

Available Commands

The CLI as a lot of commands that allow you to control your project, bellow you may available commands that it provides

  • bouer -v, --version - Display CLI version
  • bouer -h, --help - Display help information
  • bouer install - Installs dependencies for the current Bouer.js project
  • bouer create new [-t, --template ] - Create a new Bouer.js project with optional template type (blank or routing)
  • bouer create component [-p, --path ] - Create a new component in the current project
  • bouer run - Runs the development live server for the current Bouer.js project
  • bouer build [-m, --mode ] - Builds the Bouer.js project for development or production (dev or prod)
